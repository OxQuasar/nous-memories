PART B: ORACLE COMPOSITION TRACE
=================================

CAPO ARCHITECTURE
-----------------
Each LST's Aave price follows a composition chain:

  token_rate (on-chain) → CAPO cap → × market_feed (Chainlink) → USD price

The CAPO adapter:
1. Reads the on-chain exchange rate (e.g., wstETH.stEthPerToken())
2. Compares it against a stored snapshotRatio + maxGrowthRate constraint
3. Uses min(actual_rate, capped_rate) for the price calculation
4. Multiplies by the Chainlink ETH/USD feed

This means CAPO can only REDUCE the reported price, never inflate it.
It's a one-directional safety mechanism that can misfire downward.

KEY ORACLE FINDINGS
-------------------

1. CAPO IS DOWNWARD-ONLY RISK
   CAPO can only cap prices DOWN, never UP. In a real LST depeg event,
   CAPO would NOT intervene — it only fires when the exchange rate grows
   too fast (upward). For actual depegs, the Chainlink feed and DEX prices
   would flow through unimpeded.

2. REAL DEPEG → CHAINLINK LATENCY IS THE BOTTLENECK
   For a genuine DEX depeg event:
   - DEX price moves instantly
   - Chainlink stETH/ETH feed updates on deviation threshold (~0.5%) or
     heartbeat (~24h). A 3% depeg would trigger deviation-based update.
   - Aave oracle reads Chainlink → propagates to HF calculation
   - Liquidation bots execute within 1-2 blocks of oracle update
   
   Estimated total latency: 2-12 blocks (24-144 seconds) from DEX depeg
   to first liquidation. This is fast enough to catch positions.

3. CAPO SNAPSHOT STALENESS IS A SYSTEMIC RISK
   The weETH snapshot is from March 2024. If the CAPO agent misconfigures
   this token the same way it did wstETH, a similar incident could occur.
   All three tokens use 7-day snapshot delays with different max growth rates.

4. THE 2.85% INCIDENT NEARLY TRIGGERED $1.42B CASCADE
   The $1.42B wstETH position at HF 1.0398 would liquidate at 3.83% depeg.
   The incident produced 2.85% artificial depeg. If the CAPO had used a
   slightly staler ratio or the position was slightly more leveraged,
   this would have triggered a $1.42B liquidation — on a 2.85% config error,
   not a market event.

5. WHAT A REAL 3% DEPEG WOULD TRIGGER
   From our Step 1 data: wstETH has only $5.7M of DEX capacity.
   A $4.7M sell produces 2% depeg. But the $1.42B whale liquidation would
   dump ~$1.42B × 50% (max liq) = ~$710M of wstETH selling pressure.
   
   With $5.7M of DEX capacity, this is 124× the available liquidity.
   The cascade would be self-reinforcing and unstoppable.
