PERP → LENDING TEMPORAL LEAD ANALYSIS
============================================================

PHASE 1: DATA FEASIBILITY
----------------------------------------

Direct historical perp liquidation data NOT freely available:
  - Hyperliquid API: no public liquidation history endpoint
    (stats-data.hyperliquid.xyz returns 403; userFills requires
    knowing liquidated addresses; HLP vault shows deposits/withdrawals only)
  - Coinglass: requires API key for all liquidation endpoints
  - Binance: removed public forceOrders; no liquidationSnapshot in data archive
  - Binance metrics: 5-min OI snapshots available (Dec 2021 → present)

PROXY: Using Binance ETHUSDT open interest (OI) changes as liquidation proxy.
Large negative hourly OI changes during stress = forced position closures.
Significance threshold: hourly OI drop > 3%

Limitation: OI drops include voluntary closures. During stress episodes,
the forced component dominates, but the proxy is noisy for mild events.

  Proxy caveat: OI changes are an imperfect proxy for liquidations.
  Direct liquidation data (from Hyperliquid or Coinglass with API key)

  temporal ordering, however, is likely robust to proxy noise during
  major stress episodes where forced closures dominate OI changes.
