## Technical Context

### Architecture decisions

**Speak tool via ExtraTools**: Runtime injection (not global registry). Only discuss agents get it. Needs shared `speakState` reference (can't be static). Single-turn for v1. `Terminal: true` ends agent's turn. Fallback to `extractLastText` if agent doesn't call speak (backward compat with old models).

**Animation sync via centralized tick**: All panes derive `dotVisible` from `app.animTick % 2`. No per-pane state. `animRunning` bool prevents duplicate chains. Prevents desync from event-driven resets.

**Context swap on switch**: Old runLoop's captured `connCtx` must survive disconnect. New `connCtx` for new forwarding. Old context intentionally not cancelled.

**Cursor-based checkpoint**: Single source of truth (cursor, not inferred). Results stored directly, not reconstructed from session last-messages. Gate loop checkpoints identically to outer loop — no blind spot. Resume is trivial: load snapshot, seek to cursor, run.

**Proactive refresh self-schedules**: Goroutine sleeps until 10min before expiry. No cron needed. `UpdateAuth` swaps provider client in-place (concurrent-safe via SDK client replacement).

**Logos retry wraps RunTagged**: Higher-level retry (30s delays) complements provider-level retry (10s delays). Only for primary stage runs, not correction prompts. Server errors only (500+).

**Thinking block order preservation**: All blocks in `Parts` in streaming order. Text inserted before first tool_call. No separate `ThinkingBlocks` list. Matches API's original structure for signature validation.

### Patterns

**ExtraTools injection**: `ls.ExtraTools = []tools.Tool{speakTool{...}}` before RunTagged. Runner merges with agent tools (line 171-175 in `runner.go`).

**Speak consume**: `msg, done := speak.consume()` after RunTagged. Fallback: `msg = extractLastText(msgs)` if agent didn't call speak.

**Cursor checkpoint**: `setCursor(stage, round)` + `snapshot()` at every stage boundary. Resume: restore from checkpoint, `runStages` from `startIdx` with `savedGateRound`.

**Context swap**: `oldCtx, oldCancel := c.connCtx, c.connCancel; c.connCtx, c.connCancel = context.WithCancel(context.Background()); _ = oldCancel` — suppress cancel to preserve old goroutine.

**Retry wrapper**: `for attempt := 0; attempt < 3; attempt++ { err := fn(); if !isRetryable(err) { break }; sleep() }`.

**Interleaved blocks**: Accumulate all blocks (thinking, server, tool) in `Parts` as they arrive. Convert to ContentBlock array in order.

## Strategy & Approach

**Overall approach**: Fix discuss output structure (speak tool), fix UI bugs (animation, context cancel, sizing), refactor checkpoint to cursor-based (robust resume), add proactive token refresh, add server error retry, fix thinking block ordering.

**Why this approach**:

1. **Speak tool > text parsing**: Structural output prevents framing pollution, explicit stop signal replaces fragile `<done>` tag. LLM learns correct tool usage via prompting + terminal behavior.

2. **Centralized anim tick > per-pane toggle**: Eliminates race conditions from event-driven state changes. All panes blink together, simpler mental model.

3. **Context swap > shared context**: Old runLoop must survive disconnect. Swapping context before detach cleanest solution. Matches existing `Detach()` pattern.

4. **Cursor checkpoint > per-stage status**: Single source of truth (cursor), not inferred (isStageComplete heuristic). Results stored directly, not reconstructed from session last-messages. Gate loop blind spot eliminated. Resume trivial.

5. **Proactive refresh > reactive only**: Prevents workflow disruption from auth expiry during long-running bash tools. Self-scheduling based on actual expiry time simpler than cron. `UpdateAuth` swap cleaner than process restart.

6. **Logos retry > provider retry only**: Higher-level retry (30s delays) gives transient outages time to clear. Complements provider-level retry (10s delays). Server errors only to avoid burning budget on permanent failures.

7. **Preserve block order > reordering**: Signature validation tied to original position. Reordering breaks API validation. Streaming order matches API's original structure.

**Key insights**:

- **Speak tool Terminal: true ends turn cleanly** — no need for separate stop detection loop. Fallback to `extractLastText` preserves compatibility.

- **Animation desync from duplicate chains** — each `startAnimTick()` launches independent chain. Session switch triggered second launch without stopping first. `animRunning` bool prevents duplicates.

- **Context cancel propagates to detached sessions** — old runLoop's captured `connCtx` gets cancelled on disconnect. Swapping context before detach preserves old goroutine.

- **Gate loop was checkpoint blind spot** — inner loop re-ran stages but never updated checkpoint. Resume logic operated on stale data from first pass. `setCursor` at every boundary eliminates gap.

- **Cursor replaces inference** — no more "walk stages and figure out where we were". Checkpoint explicitly says where to resume. `isStageComplete` heuristic deleted.

- **Proactive refresh prevents workflow disruption** — long-running bash tools (python scripts) can exceed token TTL. Reactive refresh only triggers after failure. Proactive refresh prevents disruption.

- **Provider retry insufficient for transient outages** — 10s delays too short for server recovery. 30s logos-level retry gives outages time to clear.

- **Thinking block reordering breaks signatures** — API validates signatures against original position. Grouping thinking blocks first changes positions. Preserving streaming order maintains validation.

**Assumptions**:

- Speak tool only useful for discuss phases (solo agents don't need it)
- Animation tick rate (500ms) acceptable for all panes
- 1min idle timeout before subpane close acceptable UX
- Gate iterations ≤ 32 typical (checkpoint cursor can track)
- OAuth token refresh 10min before expiry sufficient buffer
- Server errors (500+) transient, client errors (400) permanent
- Thinking block signatures tied to position (validated empirically)

**Blockers/risks**:

- **Speak tool learning curve** — LLMs must learn to call speak. Fallback mitigates but may produce verbose transcripts if LLM outputs text + calls speak.

- **Animation sync fragility** — relies on centralized tick increment. Any code path that calls `startAnimTick()` without checking `animRunning` breaks sync again.

- **Context swap race** — if `switchSession` and `Disconnect` called concurrently, old context might get cancelled before swap. Low risk: UI is single-threaded (tea event loop).

- **Cursor checkpoint migration** — old checkpoints (per-stage status) won't resume correctly. Load succeeds but `CurrentStage=""` forces re-run from scratch. Acceptable: agent sessions preserved, just re-executes workflow.

- **Proactive refresh failure** — if `claude -p ""` fails repeatedly, token expires and workflow dies. Retry loop mitigates but eventual failure possible. Could add fallback to reactive refresh.

- **Retry budget explosion** — 9 API calls (3 logos × 3 provider) for single stage run. If all fail, significant token/cost waste. Acceptable: server outages rare, retry prevents workflow death.

- **Thinking block corruption** — if API changes signature format or validation rules, preserved order might still fail. Low risk: signature format stable since extended thinking launch.
